# リフォーム見積もり管理システム — UIサンプル

Googleフォーム＋スプレッドシート運用を置き換える、リフォーム見積もり管理システムの画面サンプル（UIモックアップ）です。「同一見積もりの修正・追跡が難しい」という課題を、見積依頼に**版（リビジョン）**をぶら下げる構造で解決する設計を、動かせる形で確認できます。

> **注意**：これはUIモックアップです。表示データはすべてダミーで、保存・送信・外部通信は行いません。画面遷移と操作感の確認用です。

## デモを開く

`index.html` をブラウザで開くと概要ページが表示され、各画面に移動できます。

## 画面一覧

| ファイル | 画面 | 内容 |
| --- | --- | --- |
| `index.html` | 概要 | 各画面への入口・設計の考え方 |
| `list.html` | 見積一覧 | ステータス絞り込み・最新版額・対応期限の自動判定 |
| `detail.html` | 見積依頼 詳細 | 依頼情報＋版履歴＋版追加・操作履歴 |
| `import.html` | 版を追加（Excel取り込み） | 見積書Excelから合計・明細を自動抽出し前版と差分 |
| `chat.html` | チャット | 見積ごとの要望のやり取り・版イベント・添付 |
| `new.html` | 新規見積依頼 | フォーム代替・物件マスタ自動入力・必須チェック |
| `diff.html` | 版の差分 | 2つの版の明細比較（追加/変更/削除/据置） |
| `users.html` | ユーザー・ロール管理 | ユーザー管理＋権限マトリクス |

## GitHub Pages で公開する手順

1. このフォルダ一式を GitHub リポジトリのルートに置いて push します。

   ```bash
   git init
   git add .
   git commit -m "Add reform estimate UI sample"
   git branch -M main
   git remote add origin https://github.com/<ユーザー名>/<リポジトリ名>.git
   git push -u origin main
   ```

2. リポジトリの **Settings → Pages** を開きます。
3. **Build and deployment** の Source を「Deploy from a branch」にし、Branch を `main` ・フォルダを `/ (root)` に設定して保存します。
4. 数十秒〜数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます。

`index.html` がルートにあるため、トップにアクセスすると概要ページが開きます。

## 設計の考え方

- **版で修正・追跡**：見積依頼の下に見積書を「版」として持たせ、修正のたびに新版を追加。過去版が残り、変更履歴と差分が追えます。
- **物件マスタ分離**：物件固有の情報はマスタに1回だけ登録。同じ物件の再見積もりは選ぶだけで物件情報が埋まります。
- **ステータスで管理**：受付〜受注／失注の進行状態と操作履歴を保持。期限超過・停滞を一覧で検知できます。

## 技術メモ

- 静的な HTML / CSS / JavaScript のみ。外部ライブラリ・ビルド不要。
- 各ページは単体で完結（インラインのCSS/JS）。
- 状態はページ内のメモリ上のみで保持し、リロードで初期化されます。

## 実運用に向けて

このサンプルは静的ホスティング（GitHub Pages）で公開できますが、実運用ではログイン・データ保存・版管理のためにサーバーサイド（例：社内サーバー上の PHP + MySQL 等）が必要です。アクセスは特定IP許可＋HTTPSでの公開構成を想定しています。GitHub Pages は不特定多数に公開されるため、ダミーデータのUIサンプルのみを置く用途に留めてください。
